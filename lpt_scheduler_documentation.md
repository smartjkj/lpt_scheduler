# 📅 LPT 배치 짭 스케줄러 (LPT Batch Job Scheduler)

이 프로젝트는 지정된 개수의 병렬 슬롯(Slot)에 여러 작업(Job)들을 **최장 처리 시간 우선(LPT, Longest Processing Time)** 알고리즘을 기반으로 할당하여 전체 소요 시간(Makespan)을 최소화하는 시뮬레이터입니다. Streamlit을 사용하여 웹 UI로 구현되었습니다.

---

## 🌟 주요 기능 (Features)

1. **사용자 맞춤형 입력 (Customizable Input)**
   - 작업 순번(ID)과 수행 시간(Time)을 쉼표(,)로 구분하여 텍스트 에어리어에 다중 입력 가능합니다.
   - 우측 사이드바에서 병렬로 처리할 **슬롯(서버/머신)의 개수**를 자유롭게 조절할 수 있습니다 (기본값: 5개).

2. **LPT 알고리즘 적용 (LPT Algorithm)**
   - 입력된 작업들을 수행 시간이 긴 순서대로(내림차순) 자동 정렬합니다.
   - 정렬된 작업들을 순회하며, 현재 누적 할당 시간이 **가장 짧은(가장 먼저 비어있는)** 슬롯에 작업을 배정합니다.
   - 이를 통해 모든 슬롯이 최대한 늦게 끝나는 시간(Makespan)을 균등화하고 단축시킵니다.

3. **헤비/라이트 작업 제약 조건 (Heavy/Light Job Constraints) - [Optional]**
   - **헤비급(Heavy) 작업**: 수행 시간이 특정 임계값(예: 100 이상)인 작업.
   - **라이트급(Light) 작업**: 임계값 미만인 작업.
   - **예약 슬롯 적용**: 
     - 5개의 슬롯 중 지정된 개수(예: 4개)는 **'헤비급 우선 할당 슬롯'**으로 동작합니다. 남아있는 헤비급 작업이 있다면 이 4개의 슬롯에만 집중적으로 들어갑니다.
     - 나머지 1개는 **'라이트급 전용 슬롯'**으로 강제되어 짧은 작업들이 대기하지 않고 즉시 처리되도록 보장합니다.
     - 단, 헤비급 작업이 모두 처리되어 더 이상 할당할 것이 없다면, 헤비급 슬롯 4개도 라이트급 작업을 받아들여 자원 낭비를 막습니다.
   - 이 규칙은 좌측 사이바에서 On/Off(체크박스) 할 수 있으며, 헤비급 판단 기준(Threshold)과 우선 할당 슬롯 개수를 동적으로 설정 가능합니다.

4. **시각화 및 비교 분석 (Visualization & Analysis)**
   - **성능 비교**: '단순 도착 순서(Baseline)' 모델과 'LPT 적용' 모델의 총 소요 시간을 나란히 보여주고, 몇 퍼센트(%)의 시간이 단축되었는지 시각적으로 알려줍니다.
   - **간트 차트 (Gantt Chart)**: `Altair` 라이브러리를 활용하여 시간의 흐름에 따른 각 슬롯별 작업 적재 상황을 가로 막대그래프로 시각화합니다. (마우스 오버 시 상세 툴팁 제공)
   - **결과 테이블 (Data Table)**:
     - 1:1 실행 순서 매핑 테이블 (원래 순서 ➔ 변경된 순서)
     - 각 슬롯별로 어떤 작업이 몇의 크기로 할당되었는지 상세 내역 표 제공.

## 🛠 코드 구조 및 로직 흐름 (Architecture)

### 1. `allocate_jobs` 함수
핵심 스케줄링 로직을 담당합니다.
* **파라미터**: 
  * `jobs`: `[{'id': str/int, 'time': float}, ...]` 형태의 작업 리스트
  * `apply_constraints`: 헤비/라이트 예약 슬롯 규칙을 적용할지 여부 (Boolean)
* **내부 로직**:
  1. `slots` 리스트 초기화 (각 슬롯의 id, total_time, jobs, heavy_count, light_count 등 추적)
  2. 제약 조건(`apply_constraints`=True)일 경우:
     * 각 슬롯이 헤비급 우선인지(인덱스 `< num_heavy_slots`), 라이트급 전용인지 판단.
     * 현재 작업이 헤비급이면 헤비급 전용 슬롯 중 가장 시간이 짧은 곳에 할당. 
       * (만약 예약된 슬롯 개수가 부족해서 못 넣으면 `unassigned_jobs`로 분류)
     * 라이트급 작업이면 1순위로 '라이트급 전용 슬롯' 중 가장 짧은 곳 탐색, 2순위로 '헤비급 우선 슬롯' 중 가장 짧은 곳 탐색.
  3. 기본 LPT(`apply_constraints`=False)일 경우:
     * 모든 슬롯 중 `total_time`이 가장 작은 곳에 그냥 밀어 넣음.
* **반환(Return)**: `(최종 Makespan, 슬롯별 상세 데이터 리스트, 할당 실패한 작업 리스트)`

### 2. Streamlit UI 흐름
1. **사이드바(Sidebar)** 세팅을 읽어와 변수(`num_slots`, `enable_constraints` 등)에 할당.
2. 텍스트 에어리어의 문장들을 파싱하여 `jobs` 리스트 생성.
3. 비교를 위한 Baseline (입력 순서 그대로) `allocate_jobs` 1회 실행.
4. 본격적인 LPT (시간순 내림차순 정렬 완료된 상태) `allocate_jobs` 1회 실행.
5. `st.metric`으로 두 방식의 결과(Makespan)와 개선율 표시.
6. 결과를 Altair Bar Chart로 변환해 화면 중단에 렌더링.
7. 작업 순번의 이동 내역과 슬롯별 배정 결과(`pandas` DataFrame)를 화면 하단에 표출.

## 🚀 실행 방법 (How to Run)

Python 환경에서 필수 라이브러리를 설치한 뒤 Streamlit 명령어로 스크립트를 실행합니다.

```bash
# 1. 필수 라이브러리 설치
pip install streamlit pandas altair

# 2. 애플리케이션 실행
streamlit run lpt_scheduler.py
```

브라우저가 열리면 좌측에 샘플 데이터를 복사해 넣고, 각종 Parameter 플래그들을 조절해가며 결과가 어떻게 달라지는지 테스트해 볼 수 있습니다.

## 최신 알고리즘 추가 반영 내역 (2026-03-16)

본 스케줄러에는 LPT 이외에도 다양한 최적화 및 메타휴리스틱 기법들이 추가로 구현되어 있습니다.

### [기본 알고리즘군]

#### 1. 단순 도착 순서 (Baseline)
- **로직:** 입력된 데이터의 순서(1번, 2번, 3번...)를 그대로 유지한 채, 가장 먼저(시간 누적이 적게) 비어있는 슬롯에 작업을 순서대로 밀어 넣습니다.
- **특징:** 어떠한 최적화도 들어가지 않은 FIFO(First-In, First-Out) 방식입니다.
- **용도:** 다른 고급 알고리즘들이 이 Baseline과 비교하여 "얼마나 시간을 단축시켰는지(개선율)"를 평가하는 절대적인 기준점 역할을 합니다.

#### 2. LPT (Longest Processing Time)
- **로직:** 모든 작업의 수행 시간을 내림차순(가장 긴 것부터)으로 정렬한 뒤, 가장 빨리 끝나는 슬롯에 순차적으로 할당합니다.
- **특징:** 전체 종료 시간(Makespan)을 줄이는 데 매우 효과적이고 널리 쓰이는 휴리스틱 기법입니다.

#### 3. SPT (Shortest Processing Time)
- **로직:** LPT와 반대로 수행 시간이 가장 짧은 오름차순으로 정렬 후 할당합니다.
- **특징:** 개별 작업이 시스템 안에 머무르는 평균 대기 시간을 최소화해야 할 때 유리한 방식입니다. 

#### 4. LPT + 헤비/라이트 예약 슬롯 (LPT_Constraint)
- **로직:** LPT 룰을 따르되, 사용자가 지정한 임계값 이상의 무거운 작업들만 들어갈 수 있는 "헤비급 전용 예약 슬롯"과, "라이트급 전용 슬롯"의 비율을 강제로 분리하여 운영합니다. 헤비급 할당이 모두 끝나면 라이트급 작업들은 제한 없이 모든 빈 슬롯을 활용합니다.

#### 5. 무작위 할당 (Random)
- **로직:** 작업들의 순서를 완전히 무작위(Shuffle)로 섞은 후 도착 순서대로 빈 슬롯에 넣습니다.

---

### [고급 알고리즘군]

#### 6. LPT + Local Search (최적화 교환)
- **로직:** 기본 LPT 방식을 수행해 일차적인 배정을 완료합니다. 이후 가장 늦게 끝나는 슬롯과 일찍 끝나는 슬롯의 작업을 반복적으로 맞바꾸며(Swap) 시간 차이가 줄어드는 방향을 탐색합니다.

#### 7. Machine Speed 기반 LPT (가중치)
- **로직:** 각 슬롯(서버)에 0.8배 ~ 1.5배의 임의 속도 가중치를 부여합니다. 단순히 누적 시간이 짧은 슬롯이 아니라 해당 슬롯의 속도로 처리했을 때 예상 완료 시간이 가장 짧은 곳으로 작업을 보냅니다.

#### 8. FFD (First Fit Decreasing / 목표치 기반 포장)
- **로직:** 작업 스케줄의 합을 슬롯 개수로 나눈 값을 '목표 평균치'로 삼고, 큰 자원부터 1번 슬롯부터 채워 넣되 목표 평균치를 초과하면 뚜껑을 닫고 다음 슬롯으로 넘어갑니다 (Bin Packing).

#### 9. LPT + Chunking (초소형 작업 묶기)
- **로직:** 일정 시간 이하의 자잘한 작업들을 하나의 커다란 가상 컨테이너 작업 `Chunk(N개)`으로 병합한 다음 전체 LPT 스케줄링에 태웁니다. 통신 지연 등 파편화 오버헤드를 막습니다.

#### 10. Simulated Annealing (모의 담금질 기법)
- **로직:** 금속을 불에 달궜다가 서서히 식히는 원리를 활용한 메타 휴리스틱. 무작위 배치를 진행하며, 전체 시간이 늘어나는 '나쁜 배치'라도 초기(온도가 높을 때)에는 일정 확률로 수용하며 지역 최적해(Local Minima)를 탈출하는 기법입니다.

---

### [동종 기계(Homogeneous) 전용 알고리즘군]

#### 11. [균등] 동적 할당 (LRFPT)
- **로직:** LPT와 유사하나, 시뮬레이션 과정에서 서버의 남은 작업(Remaining Time)을 고려해 동적으로 가장 큰 작업을 할당하는 방식입니다 (입력이 한 번에 주어질 때는 오프라인 LPT와 결과가 동일하게 수렴할 수 있습니다).

#### 12. [균등] 다중 포장 점근 (MULTIFIT)
- **로직:** 목표 한계치(최대 Makespan 예상치)를 이분 탐색(Binary Search)하며 FFD 알고리즘을 반복 수행합니다. 상자 크기를 점진적으로 줄여가며 가장 꽉 차게 들어맞는 수학적 최적의 Makespan을 수학적으로 증명(Approximation)해 냅니다.

#### 13. [균등] 최소 빈틈 채우기 (BFD)
- **로직:** 작업을 할당할 때 단순히 빈 슬롯이 아니라, "이 작업을 넣었을 때 해당 슬롯이 목표 평균치(Target Makespan)에 가장 근접하게 채워지는(빈틈이 없는)" 곳을 우선적으로 찾아 배정합니다.

#### 14. [균등] 짝수/홀수 패키징 (Round-Robin)
- **로직:** 작업을 낱개로 배정하기 전, 가장 '큰' 작업과 가장 '작은' 작업을 한 쌍(Pair)으로 묶어 패키지 덩어리로 만듭니다. 이후 그 중간 덩어리들을 LPT 룰에 태워 극단적으로 큰 작업이 한 슬롯에 몰리는 밸런스 붕괴를 사전에 통제합니다.

#### 15. [균등] 금지 목록 탐색 (Tabu Search)
- **로직:** SA나 Local Search처럼 슬롯 간 작업을 반복 교환(Swap)하되, 방금 시도했거나 실패했던 작업들의 이동 내역을 '금지 목록(Tabu List)'에 잠시 기록합니다. 같은 곳을 맴도는 무한 루프에 빠지지 않고 효율적으로 다양한 공간을 탐색하는 똑똑한 메타휴리스틱 기법입니다.

## 🏆 결론 및 시사점 (Conclusion)

총 15개에 달하는 수학적 최적화 및 메타휴리스틱 알고리즘 구동 및 **"임계값 연속 시뮬레이션"**을 통해 각종 알고리즘의 성능을 검증한 결과입니다.

결론적으로 복잡한 탐색을 거치는 알고리즘들보다 **단순 LPT (2번)** 또는 리소스 병목(DB Contention)을 회피하기 위해 슬롯 구역을 나누는 **LPT + 예약 슬롯 방식 (4번)**이 가장 실무 적용에 적합하다는 결론을 도출했습니다.

이유는 다음과 같습니다:
1. **직관적이며 예측 가능함:** 결과물(왜 이 작업이 이 슬롯에 들어갔는지)을 사람이 추적하기 가장 쉽습니다.
2. **동반 지연(Cascading Delay) 회피:** 예약 슬롯 기능을 통해 극도로 무거운 DB 자원 소모 작업들이 한꺼번에 초반에 몰려 병목을 일으키는 현상을 방어할 수 있습니다.
3. **가성비와 연산 최적화:** 복잡하게 연산 시간을 길게 끄는 메타 휴리스틱 없이도 LPT 정렬 자체가 전체 수행 시간을 단축시키는 데 압도적이고 안정적인 성능(Makespan 개선율)을 보장합니다.
