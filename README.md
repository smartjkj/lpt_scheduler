# 🚀 LPT 배치 스케줄러 애플리케이션 (LPT Batch Scheduler)

본 프로젝트는 **LPT(Longest Processing Time)** 알고리즘을 기반으로 한 배치 스케줄러 애플리케이션입니다. 작업의 효율적인 배분과 전체 종료 시간(Makespan) 최소화를 목적으로 합니다.

## 2. 🧠 알고리즘: LPT (Longest Processing Time First)

본 시스템은 **결정론적 그리디 알고리즘(Deterministic Greedy Algorithm)**인 LPT 방식을 채택하고 있습니다.

> **핵심 원리**
> *"가장 무거운 작업을 먼저 배치하여, 나중에 남는 자잘한 작업들로 유휴 시간(틈새)을 채워 넣음으로써 전체 종료 시간(Makespan)을 최소화한다."*

### ⚙️ 알고리즘 상세 동작 (Step-by-Step)
1. **정렬(Sorting)**: 수행 시간 기준으로 **내림차순** 정렬. (예: 120, 200, 45 ➔ 200, 120, 45)
2. **슬롯 초기화**: 작업을 할당할 N개의 슬롯을 생성하고 누적 시간을 `0`으로 초기화.
3. **그리디 할당**: 가장 긴 작업부터 꺼내어, 현재 **누적 시간이 가장 짧은 슬롯**에 배정.
4. **최신화 및 종료**: 할당 후 누적 시간을 업데이트하며 모든 작업이 소진될 때까지 반복.
5. **결과 도출**: 가장 늦게 끝나는 슬롯의 시간이 전체 **Makespan**이 됨.

---

## 3. 📊 알고리즘 비교 및 대안 검토

LPT는 매우 빠르지만($O(N \log N)$), 항상 절대적 최적해를 보장하지는 않습니다. 상황에 따른 대안은 다음과 같습니다.

* **대안 1. 정수 선형 계획법 (ILP)**: 수학적 수식을 통한 완전 탐색. **100% 최적해**를 보장하나, 작업량이 많아지면 연산 시간이 기하급수적으로 늘어남(NP-Hard).
* **대안 2. 유전 알고리즘 (GA)**: 진화 원리를 모방한 근사치 탐색. LPT보다 나은 해를 찾을 확률이 높으나 무작위성 제어가 까다로움.
* **대안 3. 동적 계획법 (DP)**: 메모이제이션 활용. 슬롯이 적을 때 유리하나, 슬롯이 5개 이상이면 차원의 저주로 실무 적용이 어려움.

**💡 결론:** 빠른 응답과 합리적인 성능이 필요한 실무 환경에서는 **LPT(그리디) 방식이 가장 효율적인 선택**입니다.

## 💻 4. 프로그램 핵심 구현 로직

### 4.1. 입력 파싱 (Input Parsing)
쉼표(`,`)로 구분된 텍스트를 분석하여 `float` 타입으로 변환 후 리스트화합니다.
```python
# 파싱 예시 로직
job_id = parts[0]
p_time = float(parts[1])
jobs.append({'id': job_id, 'time': p_time})
```

### 4.2. 정렬 및 스케줄링
* Timsort: Python 내장 sort()를 사용해 안정적인 내림차순 정렬 수행.
* Min 탐색: 각 슬롯의 total_time 변수를 비교하여 최적의 할당 위치 결정.

### 4.3. 예외 조건 검사 (Constraints Validation)
병목 현상 방지를 위해 Heavy/Light 작업 제한 로직을 적용합니다.
* 기준 시간을 초과하면 Heavy, 미만이면 Light로 분류.
* 특정 슬롯에 Heavy 작업이 몰리지 않도록 할당 전 카운트를 필터링합니다.

### 4.4. 비교 및 시각화 인터페이스
* Altair 간트 차트: 작업의 시작(Start)과 종료(End) 시점을 막대 그래프로 시각화.
* 1:1 매핑 비교: LPT 적용 전/후의 작업 순서를 데이터프레임으로 직접 비교 분석.
